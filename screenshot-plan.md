# Screenshot Plan

## First-Release Screenshot Plan

- Screenshot 1: Track symptoms in seconds
- Screenshot 2: Brain Fog Mode for harder days
- Screenshot 3: See patterns over time
- Screenshot 4: Review, edit, and manage history
- Screenshot 5: Create doctor-ready reports
- Screenshot 6: Private by design, with no account

## Recommended In-App Screen/State For Each Screenshot

### Screenshot 1: Track symptoms in seconds

Capture: `Log` tab / Daily Log screen.

Recommended state:

- Show the symptom toggles near the top of the form.
- Select 2-3 common symptoms such as hot flashes, poor sleep, and fatigue.
- Keep the Save button visible at the bottom.
- Use neutral demo data only.

### Screenshot 2: Brain Fog Mode for harder days

Capture: `Brain Fog Mode` screen.

Recommended state:

- Open Brain Fog Mode from Home quick action or Log toolbar.
- Select an easy-to-understand day state such as "Okay" or "Bad".
- Select 1-2 quick symptoms.
- Keep large controls and the quick-save path visible.

### Screenshot 3: See patterns over time

Capture: `Insights` tab.

Recommended state:

- Seed or create enough demo entries so the screen shows real local summaries.
- Show last-7-days entries, most frequent symptom, averages, and brain fog summary if available.
- Avoid medical certainty or diagnostic wording.

### Screenshot 4: Review, edit, and manage history

Capture: `Entry History` or `Entry Detail`.

Recommended state:

- Show the Entry History list with multiple demo entries, newest first.
- Prefer a row that displays date, overall score, top symptom, and a note preview.
- Alternate capture option: Entry Detail with Edit and Delete actions visible.

### Screenshot 5: Create doctor-ready reports

Capture: `Reports` tab / Doctor Report preview.

Recommended state:

- Use premium-unlocked test state.
- Generate a Doctor Report from demo data.
- Capture the PDF preview showing report title, date range, summary, top symptoms, and disclaimer area if possible.
- Avoid exposing any real personal health information.

### Screenshot 6: Private by design, with no account

Capture: onboarding welcome/privacy screen or Settings > Privacy & Security.

Recommended state:

- Prefer Settings > Privacy & Security if the goal is to show concrete privacy points.
- Show no account, no server, no ads, and no analytics messaging.
- Alternate capture option: onboarding "Private by design" page.

## Capture Blockers To Resolve Before Final Screenshots

- Replace any remaining placeholder support contact strategy with final App Store-ready support details if required.
- Use polished demo data that cannot be mistaken for a real user's medical information.
- Confirm premium-unlocked state for report screenshots.
- Confirm final app icon and metadata before producing final App Store screenshot set.
